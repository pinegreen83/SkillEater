# [SE-DAY1-001] 플레이어 기본 전투 루프 구축 (기본 공격)

## Background

Skill Eater의 모든 시스템은
“전투 중 경험 → 습득 → 선택” 위에 얹힌다.

그 출발점은
👉 아무 장식 없는 ‘기본 공격’이 실제 전투 루프를 형성하는 것이다.

아직:
- 스킬 습득 ❌
- 자원 ❌
- UI ❌

지금은 때리고, 맞고, 죽인다까지만 본다.

## Goal
- 플레이어가 좌클릭으로 기본 공격을 수행
- 적이 피격 → HP 감소 → 사망 흐름을 가진다
- 전투 루프가 “실제로 플레이 가능”한 상태가 된다

## Scope

포함
- 플레이어 기본 공격 1종
- 적(테스트용) 1종
- 충돌 판정
- 데미지 처리
- 사망 처리

제외 (중요)
- 스킬 시스템 전부
- 스킬 UI
- 자원(스태미나/마나)
- 보스
- 체크포인트
- 연출/이펙트/사운드

👉 ‘때릴 수 있다’만 되면 성공

## Requirements

4.1 플레이어
- 좌클릭 입력으로 기본 공격 실행
- 공격은:
- 애니메이션 or 즉시 판정 중 택1
- 형태는 단순해도 무방
- 공격 로직은:
- 추후 스킬로 확장 가능한 위치에 있어야 함
- (하드코딩 금지)

4.2 적
- 체력 보유
- 공격 피격 시:
- 데미지 적용
- 체력 감소
- 체력 0 이하 시:
- 사망 처리
- 월드에서 제거 or 비활성화

4.3 전투 루프
- 플레이어 → 공격
- 적 → 피격
- 적 → 사망

이 루프가 눈으로 확인 가능해야 한다.

## Out of Scope
- “어차피 나중에 쓸 거니까” 식의 시스템 선구현
- 스킬 데이터 구조
- 경험치, 해금, UI, 자원

👉 이 티켓은 기초 체력 검사다.

## DoD (Definition of Done)
- 좌클릭으로 공격 가능
- 적에게 데미지 적용됨
- 적이 사망함
- 코드 위치가 “기본 공격 = 스킬의 최소 단위”로 확장 가능
- 플레이하면서 “아, 전투 된다”는 감각이 듦
